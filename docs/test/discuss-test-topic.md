## 1. 스프린트 계획 회의

**참여자:** PM, 백엔드, 프론트엔드, 모바일, QA

### 주제 A
> "소셜 로그인(Google/Apple)을 이번 스프린트에 포함해야 하나?"

- velocity 40pt인데 13pt 예상, 나머지 기능과 우선순위 충돌
- 각 역할대로 팽팽하게 맞붙는 구도

### 주제 B
> "API 응답 포맷을 v2 breaking change로 바꾸는 작업, 이번 스프린트에 넣을 수 있나?"

- QA: 릴리즈 게이트 기준으로 반대
- 백엔드: ADR·SLA 근거
- PM: 출시 일정 우선

### 주제 C
> "기술 부채 해소 스프린트 — 전체 velocity 40pt를 리팩토링에만 써야 하나?"

- PM: Q3 로드맵(소셜 로그인 7월 마감) 압박으로 신규 기능 포기 불가
- 백엔드: ADR-001 위반 코드 누적, MikroORM Unit of Work 패턴 미적용으로 N+1 쿼리 폭증 → API P95 응답시간 480ms (SLA 200ms 초과)
- QA: 단위 테스트 커버리지 현재 41% (목표 80%), 이대로 릴리즈 게이트 통과 불가
- 모바일: 공유 로직 N+1 버그가 앱 크래시로 연결됨

### 주제 D
> "스프린트 진행 중 BO 긴급 요청 — DAU 추적 대시보드를 이번 스프린트에 끼워 넣자"

- PM: DAU 500명 목표 KPI 측정 수단이 없어 경영진 요청 수용 불가피
- 백엔드 + 프론트엔드: 현재 남은 스프린트 capacity 8pt, 대시보드 예상 13pt → 물리적 불가
- QA: 추가 기능 시 테스트 일정 2일 더 필요, 스프린트 리뷰 전날까지 검증 불가능

- 백엔드 ADR-002와 충돌하는 트레이드오프 논의

### 주제 C
> "실시간 알림을 WebSocket으로 구현할지 SSE로 할지"

- 백엔드: ADR-004(SSE 채택, WebSocket 미사용) 근거 제시 — 서버 상태 관리 불필요, 현재 SSE 인프라와 일관성
- 프론트엔드: WebSocket 시 TanStack Query v5 연동 불가, 클라이언트 번들 +18KB 증가 예상
- PM: 알림 기능 NPS 40 목표 달성 핵심, 8월 iOS 베타 마감 — 구현 지연 시 로드맵 전체 밀림

### 주제 D
> "검색 기능을 Elasticsearch 도입 vs PostgreSQL Full-text search로 버틸지"

- 백엔드: pgvector HNSW 이미 구축됨, pg_trgm 확장으로 혼합 검색 가능 — 별도 Elasticsearch 불필요
- PM: Elasticsearch Elastic Cloud 연간 라이선스 2,400만원, 예산 초과 CFO 승인 필요
- 프론트엔드: instant search·하이라이팅 UI 구현 비용 5pt 추가 예상, LCP 목표 2.5s 영향 검토 필요

> "앱스토어 심사 2회 연속 거부 후 — OTA(EAS Update)로 우회할 수 있는 범위가 어디까지인가?"

- 모바일: Expo EAS Update는 JS 레이어만 가능, 네이티브 브릿지·Info.plist 권한 변경은 재심사 필수

- 모바일: Android 13+ 런타임 권한(POST_NOTIFICATIONS) 동작 변경으로 전체 푸시 알림 플로우 회귀 테스트 필요
- QA: 전체 Android 기능 재검증 최소 5일 예상, 현재 스프린트 capacity 초과
- PM: 미준수 시 신규 빌드 Play Store 배포 차단 → 비즈니스 P0 리스크, 강제 진행 불가피

### 주제 A ⭐ (핵심 — 실전 배포 데이터 기반 토론)
> "RAG 문서 업로드 기능 v2.1.0 — 2026-06-25(목) 새벽 2시 배포를 예정대로 진행해야 하나?"

**개발 완료 내용:**
- BullMQ 비동기 문서 처리 파이프라인 (ADR-002 구현)
- pgvector HNSW 벡터 검색 API (text-embedding-3-large, 3072차원)
- 문서 업로드 진행률 SSE 스트리밍

**배포 일정:**
- 배포 예정일: 2026-06-25(목) 02:00
- 유지보수 공지: 2026-06-24(수) 18:00 발송 완료 (영향 범위: 문서 검색 기능 5분 중단)
- DB 마이그레이션: `document_chunks` 테이블 HNSW 인덱스 신규 추가 — 예상 소요 5분, Lock 발생으로 해당 시간 검색 API 503 응답

**현재 미해결 이슈:**
- [P1] 500MB 이상 PDF 업로드 시 BullMQ Worker 타임아웃 3분 초과 (재현율 100%, 픽스 PR #412 리뷰 중)
- [P2] 벡터 검색 결과에 코사인 유사도 0.98 이상 중복 청크 2건 반환
- [P3] 업로드 진행률 UI 소수점 표기 오류 (예: 33.333%)

**QA 상태:**
- P0: 0건 (통과)
- P1: 1건 미해결 → 릴리즈 게이트 위반 (기준: P1 2건 이하)
- 단위 테스트 통과율: 100%
- E2E 시나리오 통과율: 94% (64건 중 6건 실패, 전부 대용량 파일 케이스)

**배포 고려사항:**
- DB Lock 5분 → 헬스체크 알림 발생, 온콜 담당자 오탐 가능성
- BullMQ Redis 큐 잔여 작업 12건 → 배포 전 drain 또는 pause 필요
- 롤백 플랜: migration down 스크립트 준비됨, 소요 약 2분
- Feature Flag: `rag-upload-v2` 플래그로 500MB+ 기능 일시 비활성화 가능

**포지션 예상:**
| 역할 | 포지션 |
|------|--------|
| PM | Q3 로드맵(8월 iOS 베타) 연계, 1주 지연 시 전체 로드맵 도미노 → 강행 주장 |
| 백엔드 엔지니어 | P1 픽스 PR #412 당일 머지 가능, feature flag로 500MB+ 일시 비활성화 협상안 제시 |
| 프론트엔드 엔지니어 | v2.1.0에 프론트 변경사항 없으나 DB Lock 5분 동안 검색 API 503 응답 → 사용자 화면 오류 처리 UI 정상 동작 여부 확인 필요 |
| QA 엔지니어 | P1 미해결·E2E 94% → 릴리즈 게이트 위반, No-Go 원칙 고수 |
| 마케팅 담당자 | 25일 09:00 뉴스레터 예약 발송 완료(수신자 1,240명) — 배포 취소 시 05:00까지 철회 필요, 이후 취소 불가로 고객 혼선 및 브랜드 리스크 → 강행 압박 |
| 영업 담당자 | 2026-06-26 엔터프라이즈 A사 경영진 데모 약속(RAG 기능 시연, 예상 ARR 3,600만원) — 배포 지연 시 데모 취소·계약 클로징 3~4주 지연 → 조건부 Go 요청 |

### 주제 B
> "모바일 앱 v1.3.0 iOS/Android 동시 출시 — 앱스토어 심사 대기 중 P2 버그 발견, 어떻게 처리하나?"

**배포 배경:**
- iOS: TestFlight 베타 48시간 완료, App Store 심사 제출 완료 (예상 승인 2026-06-23)
- Android: Play Store Internal Test 완료, 2026-06-24 Production 롤아웃 20% 예정
- 신규 기능: 토론 완료 알림 푸시, 채팅 딥링크

**심사 중 발견된 이슈:**
- [P2] iOS 다크모드에서 토론 완료 알림 배지 색상 대비 2.8:1 (WCAG 기준 3:1 미달)
- [P2] Android 13 기기에서 알림 권한 요청이 첫 실행 즉시 팝업 (전략: 3일 후 기능 연계 시점)

**핵심 쟁점:**
- P2 버그를 OTA(EAS Update)로 배포 중 수정 가능한가 vs 심사 취소 후 재제출
- iOS 심사 취소 시 재심사 2~3일, 기존 TestFlight 무효화
- Android는 20% 롤아웃 시작 전에 수정 가능하나 iOS와 출시일 불일치 발생

**마케팅·영업 포지션:**
- 마케팅: iOS/Android 동시 출시 캠페인 자료 준비 완료 — 플랫폼별 출시일 불일치 시 홍보 메시지 수정 필요, 유료 광고 게재 일정 변경 비용 발생
- 영업: 파이프라인 고객 데모에서 모바일 앱 알림 기능 시연 예정 — P2 버그 노출 시 고객 신뢰 훼손 우려

### 주제 C
> "v2.0.0 → v2.1.0 배포 시 /v1/ API 병행 운영 종료를 함께 진행해야 하나?"

**현황:**
- /v1/ 엔드포인트 운영 시작: 2025-12-25 (ADR-기준 최소 6개월 병행 → 2026-06-25 종료 가능)
- 현재 /v1/ 호출 비율: 전체 API 트래픽 3.2% (모바일 앱 v1.1.x 이하 구버전 사용자)
- 종료 시: 앱 v1.1.x 이하 사용자에게 강제 업데이트 팝업 노출

**쟁점:**
- 백엔드: ADR 기준 충족, 레거시 유지 비용 vs 6개월 경과 종료 원칙
- PM: 구버전 사용자 3.2% 이탈 리스크 vs 로드맵 간소화
- QA: /v1/ 종료 후 구버전 앱 동작 검증 범위, E2E 시나리오 추가 필요
- 마케팅: 구버전 사용자 대상 업데이트 유도 이메일 캠페인 필요 — 사전 공지 없이 종료 시 CS 인입 급증 우려
- 영업: 구버전 앱 사용 중인 고객사 파악 필요 — 강제 업데이트 팝업이 계약 갱신 협상 중인 고객에게 부정적 인상 줄 가능성

### 추천 2 — 실전 배포 데이터 기반 토론 ⭐ (6자 대립 구도)

**룸:** 배포 전 최종 점검 회의 (PM·백엔드·프론트엔드·QA·마케팅·영업)

**주제:** "RAG 문서 업로드 v2.1.0 — P1 미해결 상태에서 내일 새벽 2시 배포 강행할 것인가?"

| 역할 | 포지션 |
|------|--------|
| PM | Q3 로드맵 연계, 1주 지연 불가 → 강행 |
| 백엔드 | feature flag로 리스크 격리, 당일 픽스 PR 머지 가능 → 조건부 진행 협상 |
| 프론트엔드 | DB Lock 5분 구간 503 오류 처리 UI 확인 필요 → 배포 전 스테이징 재검증 요청 |
| QA | P1 미해결·E2E 94% → 릴리즈 게이트 위반, No-Go 원칙 |
| 마케팅 | 09:00 뉴스레터 예약 발송 05:00 철회 마감 — 배포 취소 시 즉시 알려야 고객 혼선 방지 → 강행 압박 |
| 영업 | A사 경영진 데모(예상 ARR 3,600만원) 배포 지연 시 3~4주 클로징 지연 → 조건부 Go 요청 |

구체적인 날짜·버그 티켓·DB Lock·롤백 플랜·뉴스레터 마감·데모 약속 데이터가 있어 6개 에이전트가 각자 실제 수치를 근거로 논쟁하는지 검증하기 가장 적합.
